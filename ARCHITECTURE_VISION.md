## 🏗️ 現在のアーキテクチャと課題

### 現在の処理フロー

```
ユーザー入力（キーワード）
    ↓
[STEP 1] Keepa API 商品検索
    ↓
固定スコアリング（ルールベース）
    - 売上トレンド: 40点
    - 市場規模: 30点
    - 改善余地: 20点
    - 参入難易度: 10点
    ↓
Top 5 商品を表示
    ↓
[STEP 2] RainforestAPI レビュー収集
    ↓
[STEP 3] Claude AI レビュー分析
    ↓
改良案・新商品アイデアを提示
```

### 主な課題

#### 1. **検索機能の制限**
- ❌ キーワード検索のみ（条件指定不可）
- ❌ 価格帯、販売数、評価範囲などのフィルタリングなし
- ❌ ユーザーが求める商品タイプを選べない

**具体例:**
> 「安くて売れ行きが良い商品」を探したい → キーワードのみでは不可能

#### 2. **スコアリングの硬直性**
- ❌ 固定ウェイト（変更不可）
- ❌ ユーザーの戦略・優先順位を反映できない
- ❌ 「何を重視するか」を選べない

**具体例:**
> 物販初心者 → 参入難易度を重視したい（現在10点のみ）
> ベテラン → 市場規模だけを重視したい（現在30点固定）

#### 3. **Keepa判定の信頼性**
- ❌ ルールベースのみ（AI検証なし）
- ❌ 実データでの検証が不足
- ❌ 市場の複雑性を十分に捉えられていない可能性

## 🚀 理想のアーキテクチャ

### v2.0 アーキテクチャ（推奨）

```
ユーザー入力
    ├─ キーワード
    ├─ 条件フィルタ（価格帯、販売数、評価など）
    └─ スコアリングウェイト設定
    ↓
[STEP 1] Keepa API 商品検索（条件付き）
    ↓
カスタマイズ可能なスコアリング
    - ユーザー設定のウェイトを適用
    ↓
候補商品をリスト表示（Top 20〜30）
    ↓
[Claude 第一段階] Top 3〜5商品の最終チェック ⭐NEW
    - 市場トレンド適合性
    - 見落としリスク警告
    - 総合推奨度の再評価
    ↓
最終推奨商品（Top 3）
    ↓
[STEP 2] RainforestAPI レビュー収集
    ↓
[Claude 第二段階] レビュー分析
    ↓
改良案・新商品アイデアを提示
```

### 主な改善点

#### ✅ 1. 条件ベース検索

```python
# 例: ユーザーが設定できる条件
filters = {
    'price_range': (1000, 5000),        # 価格帯
    'rating_min': 3.5,                   # 最低評価
    'review_count_min': 100,             # 最低レビュー数
    'monthly_sold_min': 500,             # 最低月間販売数
    'seller_count_max': 10,              # 最大競合数
    'exclude_categories': ['本', '電子書籍']  # 除外カテゴリ
}
```

#### ✅ 2. カスタマイズ可能なスコアリング

```python
# 例: ユーザーが設定できるウェイト
weights = {
    'sales_trend': 0.40,      # 売上トレンド（デフォルト40%）
    'market_size': 0.30,      # 市場規模（デフォルト30%）
    'improvement_potential': 0.20,  # 改善余地（デフォルト20%）
    'entry_difficulty': 0.10   # 参入難易度（デフォルト10%）
}

# ユーザーは自由に調整可能
# 例: 初心者向け設定
weights_beginner = {
    'sales_trend': 0.20,
    'market_size': 0.30,
    'improvement_potential': 0.10,
    'entry_difficulty': 0.40  # 参入しやすさを最重視
}
```

#### ✅ 3. Claude AI による最終チェック（2段階方式）

**第一段階: 商品選定の最終検証**
- Top 3〜5商品に対してClaude AIが総合判断
- 見落としリスク、市場トレンド適合性をチェック
- コスト: 3〜5商品 × 小規模プロンプト = 低コスト

**第二段階: レビュー分析（既存）**
- 選定された商品のレビューを詳細分析
- 改良案・新商品アイデアを提案

## 🤖 Claude AI統合パターン比較

### パターンA: 2段階方式（推奨⭐⭐⭐）

```
Keepa判定（ルールベース）
    ↓
Top 20商品を抽出
    ↓
Claude 第一段階: Top 3〜5を最終検証
    ↓
最終推奨商品（Top 3）
    ↓
Claude 第二段階: レビュー分析
```

**メリット:**
- ✅ コスト効率が良い（AIは最終チェックのみ）
- ✅ 高速（ルールベースで大部分を処理）
- ✅ 品質と速度のバランスが最良

**デメリット:**
- ⚠️ 実装がやや複雑

**コスト見積もり:**
- 1検索あたり: ¥10〜30（Claude API）

### パターンB: 警告のみ方式（⭐⭐）

```
Keepa判定（ルールベース）
    ↓
Top 5商品を表示
    ↓
Claude: 各商品の「要注意ポイント」のみ指摘
```

**メリット:**
- ✅ 実装が簡単
- ✅ コストが低い

**デメリット:**
- ❌ AIの活用が限定的
- ❌ スコアリング精度は向上しない

### パターンC: 完全AI判定（⭐）

```
Keepa API で商品データ取得
    ↓
Claude AI が全商品を評価・ランキング
    ↓
Top 5を表示
```

**メリット:**
- ✅ 理論上は最高精度

**デメリット:**
- ❌ コストが高い（1検索あたり¥100〜300）
- ❌ 処理が遅い（1分以上かかる可能性）
- ❌ ユーザーの待ち時間が長すぎる

### 推奨: パターンA（2段階方式）

**理由:**
1. コストパフォーマンスが最良
2. 処理速度とAI品質のバランスが良い
3. ユーザーは数秒で結果を得られる
4. AIの「最終判断」という役割が明確
