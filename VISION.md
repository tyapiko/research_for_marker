## 📊 データ収集の拡張（API制約解除後）

### 1. 商品検索の深化

**現状の課題**:
- RainforestAPIで上位10件のみ取得
- Amazon検索1ページ目しか見ていない
- スポンサー広告（大手）が上位を占める

**理想の姿**:
```python
# 複数ページ + 複数ソート順で網羅的に取得
search_configs = [
    {'page': 1, 'sort': 'featured'},      # おすすめ順
    {'page': 2, 'sort': 'featured'},
    {'page': 1, 'sort': 'review_rank'},   # レビュー評価順
    {'page': 1, 'sort': 'price_asc'},     # 価格安い順
    {'page': 1, 'sort': 'newest'},        # 新着順
]
# → 合計50-100件を取得して分析
```

**効果**:
- ✅ 見逃しリスク削減：11位以降の「隠れたチャンス商品」を発見
- ✅ 多角的評価：価格帯別・評価別の市場構造を把握
- ✅ 新興ブランド発見：新着順で後発参入企業を追跡

**コスト**: RainforestAPI約5倍、Keepa API約5倍（月¥5,000-10,000増）

### 2. レビュー収集の完全版

**現状の課題**:
- productエンドポイントのtop_reviewsのみ（10-20件）
- 低評価レビューが不足する可能性
- レビュー全体の傾向が掴めない

**理想の姿**:
```python
# RainforestAPI有料プランで全レビュー取得
reviews = collector.collect_all_reviews(
    asin=asin,
    rating_filter=[1, 2, 3],  # 低評価のみ
    verified_only=True,        # 認証済み購入のみ
    max_count=500              # 最大500件
)
```

**追加機能**:
- ⭐ **レビュー時系列分析**: 最近3ヶ月の低評価レビュー急増を検知
- ⭐ **画像付きレビュー優先**: ユーザーが撮影した実物写真から問題を把握
- ⭐ **返品率データ**: Keepa APIの返品率データと組み合わせ

**効果**:
- ✅ Claude分析の精度向上：★1-2のレビューを確実に収集
- ✅ 最近の問題発見：製品改悪や品質低下をリアルタイム検知
- ✅ 具体的な不満把握：写真付きレビューで視覚的に問題点を理解
